React Forms

Forms work a little different than they do in vanilla HTML. Forms in React naturally keep some internal state, so that you can access information from the inputs.

So it is convenient to have a JS function that handles the submission of the form and has access to the data the user entered. The technique to get this is called a controlled component. React really wants to override the vanilla HTML such that React has control over the form and syncs the state up. React controls what is shown in the component and what happens when the user types (which gets kept in state).

This way there is no gap of knowledge between React state and the form input - React state is the source of truth. With a controlled component, every state mutation will have an associated handler function. This makes it easy to modify or validate user input.

Using UUID for Unique Keys

Sometimes there isn't a great natural id to use for the key prop that React needs to keep arrays and whatnot ordered correctly. In that scenario, one option is to install uuid, a library that will generate a universally unique id for you. Install with npm install uuid
