React Context

In a typical React app, data is passed top-down via props, but this is cumbersome for certain types of props that are required by many components within an application. Context provides a way to share values between components without having to explicitly pass a prop through every level of the tree.

Context is designed to share data that can be considered 'global' for a tree of React components, such as the current authenticated user, them or preferred language.

Context can also be used (with some help from useReducer) to solve the issue we saw in the last section, in which some of our hooks were passed through multiple components, in which they were not ever used, and finally used way down the chain. Its an organizational issue. You are going to see that component recieve a prop and imagine it is important for that component, when in fact it is just passing through.

There are some conflicting ideas about if context should be used to solve the above issue.
