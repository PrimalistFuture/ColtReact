React Props

Intro

Properties or Props allow us to make a component more customizable and more reusable. Props are how we pass data from one component to another, from a parent to a child.

Data is passed into a component like you would specific a attribute in HTML propertyName="Whatever".

From there, the component must know how to handle the data passed into it, otherwise nothing happens.

One crucial point about props is that they are immutable - they cannot be changed, altered or add to it; they are read only. Data can of course be changed, but that is not done with props, that should be done with State.

The data passed into a component as a prop can be of any type, but strings should be double quoted and all other types need to be surrounded with curly brackets.

Styling React

Generally when styling React components, the .css file will have the same name as the component it is stying and it will only govern the styling of that component and nothing else.

Generally that component will have a class in a div of the name of the component. But instead of calling it class like you would in normal css and html, call it className.

Additionally, when styling with a label attribute, "for" is reserved and must now be "htmlFor" instead.

We can instead of a dedicated .css file style in-line as well. This will be done with a "style" attribute and then we can pass in some javascript with {}.
