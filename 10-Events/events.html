React Events

Commonly Used React Events

You can attach event handlers to HTML elements in React via special reserved attributes like onClick. You can do this in vanilla JavaScript too but the syntax is a bit different.

Some common ones: onClick, onMouseOver, onSubmit, onKeyDown, onKeyUp, onKeyPress. There are a ton.

Method Binding

We must bind a method because the keyword this loses context within our event handlers - they aren't sure what this is. We already saw one way of binding a method in the constructor - which should be considered the default. But there are more.

Inline can be done within the event attribute. It is very explicit, but if that event handler function is part of multiple components, it must be bound every time. This means that a 'new' function is created on every render. Mostly these are small points, but they could matter.

It can also be done inline with an arrow function - syntax is a little different

There is another 'experimental' way of binding that works fine in React because of Babel, but it is not in real JavaScript yet. Colt has a video on it.

Method Binding with Arguments
