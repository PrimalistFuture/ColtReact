React

The main idea of React is to make components, that encapsulate logic and HTML in a class, which makes it easier to build modular applications.

In general, you want your JS, CSS, and HTML to all be seperate. React does this in a slightly different way, where each component has those three parts, but combined into one piece.

Front End Frameworks

A library is some code that can be incorporated into your code. As opposed to a framework which is some code that you can use to build your own code. Frameworks are opinionated, they want things done their way - like files need to be in a certain spot and named a certain way.

There are many popular front end frameworks, like angular, ember, and vue. Angular and the like come with a lot of features out of the box, like testing, a client router, command line interface and tons more.

React refers to itself as a library. The vanilla version is far less feature rich than something like Angular. However there are bunch of tools that near standards like React Router, Webpack, Redux and more, which pushes the React development pattern towards the framework end.

Components

Components are the building blocks of react. Components let you split the UI into indepenent, reusable pieces, and think about each piece in isolation. Conceptually, components are like JS functions. They accept arbitary inputs, called props, and return React elements describing what should appear on the screen.
