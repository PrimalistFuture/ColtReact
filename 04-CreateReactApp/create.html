Create React App

We have seen that React is a front-end library, and you don't need to get any fancy server-side stuff. We have thus far gotten our React from a CDN that we have imported with script tags and have then transpiled into real JavaScript with JSX in the browser at runtime.

Create-React-App is a utility script that creates a skeleton react project, and sets it up so that JS files are run through Babel automatically. Lets us use super-modern JS features and idioms, and makes testing and deployment much easier.

It is a great way to get React up and running.

Webpack

CRA is built on top of Webpack which is a JS utility that enables module importing and exporting, packages up all of our css/images/js into a single file for the browser, has 'hot-reloading' so we don't have to restart server to see changes, enables easy testing and deployment.

Generally, webpack is a pain to get up and running, but CRA does all of that setup for us.

Installing Create-React-App

There are a few options. I will use this one. It usually takes a bit.

npx create-react-app app-name-here

Creating a New App

Create React App which will make a bunch of stuff for you. In the public folder, it will start you off with and index.html. Most of your time will be spent in the src folder.

Starting and Stoping the Server

npm start in the command line from inside of the application.

Then use control-c to stop your server.

Modules Crash Course

This isn't specific to React or create-react-app, but now that we are in CRA, we will be importing and exporting files so I think a refresher will be good.

CRA uses ES2015 "modules", which is newer standardized version of Node's require(). You use this to export / import classes / data / functions between JS files.

Modules are files that contain code you want to use somethere else. Modules are imported at the top of the file you want to use them in,

Check the demo and helpers for useful comments.

When to use export default

The general case for exporting something as default is when it is the main thing of that file, and will almost certainly always be needed for import.
